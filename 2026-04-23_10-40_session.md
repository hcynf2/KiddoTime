# KiddoTime Dev Session — 2026-04-23

## App Overview

KiddoTime is an Android parental control app (Jetpack Compose + MVVM, Material 3).
Two modes selected at launch: **Parent** and **Child**.

### Core features (as of this session)
- App usage monitoring via `AppMonitorService` (foreground service, polls every 1 s)
- Per-app and total daily screen time limits
- Overlay games (Card Matching, Clean-Up, What's Next?) triggered when a limit is hit
- Lock screen with parent PIN after game completion
- Bedtime scheduling with auto-lock/unlock
- Child dashboard: stars, streaks, badges, today's progress
- Parent dashboard: 10 action-button grid, each opening a bottom sheet
- Time-extension requests from child, approved by parent via PIN
- Data privacy screen: export and full data deletion

- `ParentScreen` `TimeRequestsSheetContent` — toggle card at top; analytics section at bottom (stat header, approval rate bar, avg response time, most requested app); only shown when `totalRequests > 0`

### 3 — Permission Re-check Bug Fix

**Problem:** After tapping "I'm a Parent" and granting Usage Access in Settings, the app stayed on the permission prompt forever. `LaunchedEffect(Unit)` fires only once on composition; no re-check happened on return from Settings.

**Fix:**
- `ParentViewModel.recheckPermission()` — checks `hasPermission` guard to prevent duplicate flow launches, then calls `loadUsageStats()` if permission is now granted
- `ParentScreen` — `DisposableEffect(lifecycleOwner)` registers a `LifecycleEventObserver` that calls `recheckPermission()` on every `ON_RESUME`
